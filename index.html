História da web

(Voltar)

A internet surge na década de 60 durante a guerra fria, o governo norte americano pretendia criar uma rede de comunicação entre seu computadores com o objetivo de melhorar a comunicação entre os mesmos e ser mais eficiente.

Também, os primeiros computadores surgiram um pouco antes, tendo o seu primeiro como o Mark 1 em 1944. Na época esse computador era usado para cálculos e funcionava a base de várias válvulas e eram de tamanhos de quartos e pesavam algumas toneladas.

Exemplos de computadores:

  1. ENIAC, primeiro computador eletrônico;
  2. NPL Pilot ACE, primeiro computador com múltiplos própositos;
  3. Apple-1, primeiro computador da apple e um dos primeiros computadores pessoais da história.

O que são clients

(Voltar)

Client pode ser resumido como o consumidor, literalmente como o nome sugere, o cliente. Os clients são as máquinas que fazem as requisições aos servidores, ao acessar um site por exemplo, o client será aquele que fará a busca e visitará o site, é aquele que busca a informação ou que consome o que é oferecido pelo site.

Clients não armazenam arquivos de um site, como um banco de dados, somente armazenam dados que esteja na máquina

O que são servers

(Voltar)

Servers se comunicam com os clients, porém do lado oposto da internet, os clients servem para armazenar dados e/ou fornecer esses dados ao clieent. Existem várias formas de servers, como os bancos de dados, que armazenam informações de usuários, arquivos de uma empresa e outros exemplos.

Clients e servers utilizam diferentes linguagens de programação. HTML por exemplo é uma linguagem lida pelo client, mas não utilizada diretamente no server (vale lembrar que HTML não é uma linguagem de programação, mas sim de marcação).

Linguagens de programação

(Voltar)

Pegando gancho, linguagens de programação, são códigos utilizados para dar comandos ao computador para que se alcance um objetivo. Como dito anteriormente as linguagens usadas nos clients e nos servers são diferentes, porém, existem algumas que funcionam nos dois, como por exemplo o JavaScript, que pode ser lido tanto por clients quanto por servers, porém precisam ser compiladas e lidas por um programa localizado no browsers, sistema de busca, o mais famoso é o V8, sistema do google chrome.

Vale também mencionar que sistemas operacionais são diferentes em cada um também, porém nesse caso não há sistemas operacionais idênticos para servers e clients.

Estrutura básica do HTML

(Voltar)

HTML, ou linguagem de marcação de hipertexto é a linguagem utilizada para construção de sites web em geral, portanto essa linguagem precisa ser simples e didática para que seja facilmente entendida por programadores e estudiosos das áreas. Portanto, podemos definir o básico do HTML em duas partes:

  1. Head;
  2. Body

Como o nome sugere, Head é a cabeça do site, é tudo que o usuário não ve, parte dedica ao programador para fazer algumas formatações e informar ao browser o nome, onde está hospedado e a linguagem do site, basicamente é isso.

Já o body é o que o usuário consegue ver e ler, claro que a parte das tags não pode ser vista pelo usuário, a não ser que o mesmo utilize da função "inspecionar o elemento".

A inspeção de elementos é um mecanismo disponibilizado pelos browsers para que se possa ler a linguagem de marcação e programação utilizada no site, e é possível que parte dos códigos sejam bloqueados por parte do servidor, para evitar problemas com segurança.

O que são tags

(Voltar)

Tags são os comandos dados ao computador para que ele execute uma ação. As tags usados no HTML sempre serão iniciadas pelo símbolo "<" e fechadas por ">", algumas tags serão fechadas por "/>" e a maioria delas sera fechadas pelo seu próprio nome, porem com a barra vindo antes: <p>Teste</p>.
